      # let / const / var 的区别

      JS中声明变量或者函数的方式

      • 【传统】
        • var n = 10;
        • function func(){} -> var func=function(){};
      • 【ES6】
        • let n = 10;
        • const m = 20;
        • let func = ()=>{};
        • import xxx from 'xxx';

      const设置的是常量，存储的值不能被改变 【不对】
      const创建的变量，它的指针指向一旦确定，不能再被修改【正确】
      let设置的是变量，存储的值可以改变 【正确】

      const n = 10;
n = 20; //=>Uncaught TypeError: Assignment to constant variable.
console.log(n);

      # let 和 var 的区别？

      • => let不存在变量提升
      • => let不允许重复声明
      • => let会产生块级作用域
      • => 暂时性死区的问题

      变量提升：在当前上下文代码自上而下执行之前，会把所有带var/function关键字的进行提前的声明或者定义（带var是只声明，带function是声明+定义(赋值)都完成了）

      EC(G) 变量提升: var a; func=AAAFFF000; 代码执行：

      console.log(a); //=>undefined
func(); //=>"OK"
var a = 12;

function func() {
    console.log('OK');
}

      # this的几种基础情况

      • 事件绑定
      • 函数执行
        • 匿名函数
        • 成员访问
        • 普通函数
      var x = 3,
    obj = {x: 5};
obj.fn = (function () {
    this.x *= ++x;
    return function (y) {
        this.x *= (++x)+y;
        console.log(x);
    }
})();
var fn = obj.fn;
obj.fn(6);
fn(4);
console.log(obj.x, x);
