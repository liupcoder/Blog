# let / const / var 的区别

JS中声明变量或者函数的方式

  • 【传统】
    • var n = 10;
    • function func(){} -> var func=function(){};
  • 【ES6】
    • let n = 10;
    • const m = 20;
    • let func = ()=>{};
    • import xxx from 'xxx';

const设置的是常量，存储的值不能被改变 【不对】
const创建的变量，它的指针指向一旦确定，不能再被修改【正确】
let设置的是变量，存储的值可以改变 【正确】

const n = 10;
n = 20; //=>Uncaught TypeError: Assignment to constant variable.
console.log(n);

# let 和 var 的区别？

  • => let不存在变量提升
  • => let不允许重复声明
  • => let会产生块级作用域
  • => 暂时性死区的问题

变量提升：在当前上下文代码自上而下执行之前，会把所有带var/function关键字的进行提前的声明或者定义（带var是只声明，带function是声明+定义(赋值)都完成了）

# 块级作用域

前端代码中的上下文（作用域）

  1. 全局上下文
  2. 函数执行形成的私有上下文
var a = 12;
if (1 == 1) {
  console.log(a); //12
  var a = 13;
  console.log(a); //13
}
console.log(a); //13
